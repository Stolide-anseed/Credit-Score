# Описание признаков датасета Credit Score

Файл описывает колонки из `data/train.csv`: что показывает каждый признак и как его можно понимать при подготовке модели кредитного скоринга.

| Признак | Что показывает |
| --- | --- |
| `ID` | Уникальный идентификатор строки наблюдения. Технический признак, обычно не используется для обучения модели. |
| `Customer_ID` | Уникальный идентификатор клиента. Нужен для группировки записей одного клиента по месяцам и восстановления пропусков внутри истории клиента. |
| `Month` | Месяц наблюдения по клиенту. В train-данных у каждого клиента несколько месячных записей. |
| `Name` | Имя клиента. Персональный идентификатор, обычно исключается из модельных признаков. |
| `Age` | Возраст клиента. В данных встречаются некорректные значения, поэтому признак требует очистки. |
| `SSN` | Номер социального страхования или другой персональный идентификатор клиента. Обычно исключается из обучения. |
| `Occupation` | Профессия или сфера занятости клиента. Может быть полезна как категориальный признак после очистки пропусков и кодирования. |
| `Annual_Income` | Годовой доход клиента. Показывает общий уровень дохода за год. |
| `Monthly_Inhand_Salary` | Ежемесячный доход клиента на руки после вычетов. Используется как более прямой показатель платежеспособности. |
| `Num_Bank_Accounts` | Количество банковских счетов клиента. Может отражать финансовую активность и сложность финансового профиля. |
| `Num_Credit_Card` | Количество кредитных карт клиента. Может быть связано с кредитной нагрузкой и кредитным поведением. |
| `Interest_Rate` | Процентная ставка по кредитным продуктам клиента. Более высокие значения часто связаны с более рискованным кредитным профилем. |
| `Num_of_Loan` | Количество кредитов у клиента. Показывает уровень кредитной нагрузки. |
| `Type_of_Loan` | Типы кредитов клиента, перечисленные строкой через запятую. Можно преобразовывать в отдельные индикаторы типов кредитов. |
| `Delay_from_due_date` | Количество дней задержки платежа относительно даты оплаты. Чем больше задержка, тем хуже платежная дисциплина. |
| `Num_of_Delayed_Payment` | Количество просроченных платежей. Показывает частоту нарушений графика платежей. |
| `Changed_Credit_Limit` | Изменение кредитного лимита клиента. Может отражать пересмотр банком доступного лимита. |
| `Num_Credit_Inquiries` | Количество запросов кредитной истории или кредитных проверок. Частые запросы могут указывать на активный поиск новых кредитов. |
| `Credit_Mix` | Качество или разнообразие кредитного портфеля клиента. В данных принимает значения вроде `Bad`, `Standard`, `Good`. |
| `Outstanding_Debt` | Текущая непогашенная задолженность клиента. Показывает остаток долговой нагрузки. |
| `Credit_Utilization_Ratio` | Доля использованного кредитного лимита. Высокое значение может означать, что клиент сильно использует доступный кредит. |
| `Credit_History_Age` | Возраст кредитной истории клиента в формате лет и месяцев. В preprocessing удобно переводить в количество месяцев. |
| `Payment_of_Min_Amount` | Платит ли клиент минимальный обязательный платеж. Может отражать стиль обслуживания долга. |
| `Total_EMI_per_month` | Общая сумма ежемесячных платежей по кредитам. EMI означает регулярный ежемесячный платеж. |
| `Amount_invested_monthly` | Сумма, которую клиент инвестирует ежемесячно. Может показывать финансовый запас после расходов и платежей. |
| `Payment_Behaviour` | Поведенческий паттерн платежей и трат клиента, например сочетание низких или высоких трат с размером платежей. |
| `Monthly_Balance` | Остаток средств клиента за месяц после доходов, платежей, расходов и инвестиций. |
| `Credit_Score` | Целевая переменная: рейтинг кредитоспособности клиента. В данных принимает значения `Poor`, `Standard`, `Good`. |

## Практические замечания

- `ID`, `Name` и `SSN` лучше не использовать как модельные признаки, потому что это идентификаторы или персональные данные.
- `Customer_ID` полезен на этапе preprocessing для восстановления пропусков по истории клиента, но обычно удаляется перед обучением модели.
- Часть числовых признаков хранится строками и содержит служебные символы, поэтому перед обучением их нужно преобразовать в числа.
- В данных встречаются пропуски, маркеры вроде `_` и аномальные значения, поэтому признаки требуют очистки перед кодированием и масштабированием.
